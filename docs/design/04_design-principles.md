# Princípios De Design

1. Uma ação principal por tela ou estado.
2. O conteúdo mais importante aparece primeiro.
3. A hierarquia deve funcionar antes da cor.
4. Use poucas cores simultaneamente.
5. Reserve accent para ação e foco.
6. Muito espaço negativo, principalmente entre grupos.
7. Pouco texto, sempre específico.
8. Cards devem respirar; não compactar por padrão.
9. Uma borda deve separar superfícies com motivo claro.
10. Uma sombra deve indicar elevação real.
11. Não usar gradientes como substituto de identidade.
12. Ícones são outline e pertencem a uma família única.
13. Ícones não devem carregar significado crítico sozinhos.
14. Estados devem combinar texto, forma ou ícone com cor.
15. Ação destrutiva deve ser visualmente distinta e confirmada.
16. Ações críticas nunca ficam escondidas sem alternativa visível.
17. O próximo passo deve ser evidente.
18. A interface deve preservar o contexto após uma ação.
19. Feedback aparece perto do elemento que mudou.
20. Loading deve ocupar o espaço do conteúdo esperado.
21. Empty state deve explicar e orientar.
22. Erro deve explicar causa e recuperação.
23. Confirmação deve descrever consequência.
24. Labels visíveis são obrigatórios em formulários.
25. Placeholder é exemplo, não instrução essencial.
26. Títulos usam sentence case.
27. O produto usa termos consistentes entre perfis.
28. Um H1 representa a tarefa da página.
29. A escala tipográfica define importância, não caixa alta.
30. Contraste vem antes de sutileza estética.
31. Foco de teclado é parte do design, não detalhe técnico.
32. Áreas clicáveis devem ser confortáveis no touch.
33. Mobile preserva a tarefa, não necessariamente a composição desktop.
34. Tabelas priorizam leitura antes de quantidade de colunas.
35. Filtros devem revelar o que está aplicado.
36. Tabs agrupam alternativas do mesmo nível.
37. Dialog interrompe somente quando a decisão exige atenção.
38. Drawer preserva contexto quando a tarefa é complementar.
39. Motion comunica mudança, não enfeita.
40. Motion não deve bloquear trabalho.
41. Respeitar `prefers-reduced-motion` sempre.
42. Superfície opaca é padrão para dados sensíveis.
43. White label personaliza marca, não altera semântica de status.
44. Branding não pode reduzir legibilidade.
45. O visual público e o painel pertencem à mesma marca.
46. A densidade acompanha o perfil e a frequência da tarefa.
47. A informação clínica recebe tratamento discreto e claro.
48. A interface deve parecer estável em todos os estados.
49. Consistência vence preferência local de uma tela.
50. Toda decisão visual deve reduzir dúvida operacional.
